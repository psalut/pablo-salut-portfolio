## 🧱 Architecture & Approach

The project follows a **component-driven architecture**, separating concerns into:

* **Sections** → Main blocks of the page (Hero, Projects, Experience, etc.)
* **UI Components** → Reusable elements (cards, buttons, tags)
* **Content Layer** → Structured data (projects, experience, skills)
* **Layout System** → Global structure (Navbar, Footer, Layout)

This allows:

* scalability
* easy content updates
* clean separation between data and presentation

---

## 🛠️ Tech Stack

### Core

* **Astro** → Static-first framework for performance and SEO
* **TypeScript** → Type safety and maintainability
* **Tailwind CSS** → Utility-first styling for rapid UI development

### UI & Interactivity

* **React (Astro Islands)** → Used only where interactivity is required
* **Lucide Icons** → Clean and consistent icon system

### Deployment

* **Vercel** → Fast and optimized hosting platform

---

## ⚡ Why Astro?

Astro was chosen because:

* 🚀 Minimal JavaScript by default
* ⚡ Excellent performance (static-first rendering)
* 🧩 Partial hydration (only interactive components use JS)
* 🧼 Clean separation between logic and markup
* 🔍 SEO-friendly out of the box

This makes it ideal for a **portfolio project focused on speed and clarity**.

## 📂 Project Structure

```bash

    ui/
  content/
  layouts/
  pages/
  styles/
```

* `sections` → Main page blocks
* `ui` → Reusable components

* `layouts` → Global structure
